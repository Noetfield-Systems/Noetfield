How it is produced

  1. Evaluate — policy decision with reason_code and policy_refs tied to a Request ID
  2. Record — entries append to the governed audit log (integrity hash per event)
  3. Export — audit-export by RID for engagement intake (Governance API reference)
  4. Trust Ledger API — live on platform: POST /api/v1/tle/draft, multi-approve, GET /api/v1/tle/{id}/export, connector POST /api/v1/connectors/{id}/sync (OpenAPI at /docs when platform is running)
  5. Workspace — read-only list and detail at Governance Console · Trust Ledger (local dev: http://localhost:13080/trust-ledger)
